# پنهان‌نگاری تصویر - مخفی و جستجو

[English](README.md) | **فارسی**

یک برنامه دسکتاپ مدرن پایتون برای پنهان‌نگاری تصویر. پیام‌های رمزنگاری شده را در تصاویر با استفاده از رمزگذاری LSB و رمزنگاری AES-256 مخفی کنید.

## امکانات

- **رمزنگاری LSB** - پیام‌ها را در بیت‌های کم‌اهمیت پیکسل‌ها مخفی کنید
- **رمزنگاری AES-256** - رمزنگاری اختیاری Fernet با مشتق‌سازی کلید PBKDF2
- **نمک یکتا** - هر پیام یک نمک تصادفی یکتا برای حداکثر امنیت دریافت می‌کند
- **پردازش دسته‌ای** - چندین تصویر را همزمان رمزگذاری/رمزگشایی کنید
- **کشیدن و رها کردن** - تصاویر را مستقیماً روی برنامه رها کنید
- **فرمت‌های چندگانه** - خروجی PNG، BMP یا TIFF (همه بدون اتلاف)
- **پوسته تاریک** - رابط کاربری الهام گرفته از گیت‌هاب با انیمیشن‌های نرم
- **اندازه‌گیر خطر** - تحلیل خطر تشخیص در زمان واقعی
- **مقایسه تصویر** - نمایش در کنار هم با محاسبه PSNR

```bash
pip install customtkinter Pillow cryptography numpy

```

## نحوه کار

هر پیکسل دارای کانال‌های R، G، B (0-255) است. رمزگذاری LSB بیت کم‌اهمیت هر کانال را با یک بیت از پیام شما جایگزین می‌کند. تغییر بیت 0 مقداری مانند 182 (10110110) به 183 (10110111) تفاوتی برابر 1 ایجاد می‌کند - برای چشم انسان نامرئی.

### پروتکل دودویی

```
MAGIC (32 بیت) | FLAG (8 بیت) | SALT (128 بیت) | LENGTH (32 بیت) | PAYLOAD (N بیت)
```

- **MAGIC**: هدر `"STEG"` برای تشخیص
- **FLAG**: `0xFF` اگر رمزنگاری شده، `0x00` اگر متن ساده
- **SALT**: نمک تصادفی یکتا برای مشتق‌سازی کلید
- **LENGTH**: اندازه بار در بیت
- **PAYLOAD**: پیام مخفی (اختیاراً رمزنگاری شده)

## کلیدهای میانبر

- `Ctrl+O` - باز کردن تصویر
- `Ctrl+S` - ذخیره نتیجه
- `Ctrl+E` - رمزگذاری پیام
- `Ctrl+D` - رمزگشایی پیام
- `Ctrl+V` - چسباندن تصویر از حافظه
- `Escape` - پاک کردن نمای فعلی

## محدودیت‌ها

- تصاویر JPEG داده‌های مخفی را از دست می‌دهند (فشرده‌سازی اتلافی LSBها را از بین می‌برد)
- ویرایش، تغییر اندازه یا برش تصویر داده‌های مخفی را از بین می‌برد
- بدون رمز عبور، هر کسی می‌تواند پیام را رمزگشایی کند
- پنهان‌نگاری LSB توسط تحلیل آماری قابل تشخیص است

## نکات امنیتی

- تمام پردازش‌ها 100% محلی است - هیچ چیزی از رایانه شما خارج نمی‌شود
- رمزنگاری از استاندارد صنعتی AES-256 از طریق Fernet استفاده می‌کند
- PBKDF2 با 100,000 تکرار برای مشتق‌سازی امن کلید
- هر پیام یک نمک تصادفی یکتا دریافت می‌کند
- خروجی همیشه بدون اتلاف است (PNG، BMP، TIFF)
